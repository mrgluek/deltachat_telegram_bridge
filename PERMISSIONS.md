# Отчет по правам доступа (Permissions Report)

Текущая модель прав доступа в боте разделяет полномочия на две основные ветки: **Группы (Bridges)** и **Каналы (Channels)**. Также логика кардинально меняется в зависимости от того, задан ли в настройках (`.env`) Владелец бота (`admin_tg_id` / `admin_dc_email`).

---

## 1. Управление Группами (Bridges)

Группы — это двусторонние мосты между чатом в Telegram и чатом в Delta Chat.

### На стороне Telegram

Команды (`/bridge`, `/unbridge`) работают только внутри самих групп Telegram.

* **Если Владелец бота задан (`admin_tg_id`):**
  * **Создание моста (`/bridge`):** Доступно **только** Владельцу и добавленным им Суб-админам (через `/adminadd`).
  * **Удаление моста (`/unbridge`):** Владелец может удалить любой мост. Суб-админы могут удалить **только те мосты, которые они сами создали**. Обычные админы TG-группы прав не имеют.
* **Если Владелец бота НЕ задан (Публичный режим):**
  * Любой администратор Telegram-группы (или ее создатель) может привязать или отвязать свою группу.

### На стороне Delta Chat

* Нет публичных списков групп. Привязка создает новую группу в Delta Chat. Любой пользователь, которого добавили в эту DC-группу, может читать и писать сообщения, которые будут пересылаться в Telegram. Специфичных команд администрирования групп изнутри DC сейчас нет.

---

## 2. Управление Каналами (Channels)

Каналы — это односторонняя трансляция (TG → DC).

### На стороне Telegram

Команды (`/channeladd`, `/channelremove`, `/channels`) работают **только в личных сообщениях (private chat)** с ботом.

* **Если Владелец бота задан (`admin_tg_id`):**
  * **Создание трансляции (`/channeladd`):** Доступно Владельцу и назначенным им Суб-админам.
  * **Удаление трансляции (`/channelremove`):** Владелец может удалить любой канал. Суб-админы могут удалить **только те каналы, которые они сами добавили**.
  * **Просмотр (`/channels`):** Владелец видит все каналы, суб-админы видят только свои.
* **Если Владелец бота НЕ задан (Публичный режим):**
  * Любой пользователь может добавить канал. Однако при просмотре (`/channels`) или удалении (`/channelremove`), пользователь будет иметь доступ **только к тем каналам, которые добавил он сам**.

### На стороне Delta Chat

Пользователи DC могут запрашивать список каналов командой `/channels` и присоединяться к ним через `/channel<N>`.

* **Для DC-Администратора (совпадение email с `admin_dc_email`):**
  * Видит полный список всех привязанных каналов.
  * Может запрашивать QR-коды и ссылки-приглашения (join links) абсолютно для **любых** каналов (как публичных, так и приватных).
* **Для обычных DC-пользователей:**
  * Видят в списке **только публичные каналы** (те, которые были добавлены по `@username`).
  * **Не видят и не могут присоединиться** к приватным каналам (которые были добавлены по ID вида `-100...` или через приватные ссылки). Бот выдаст ошибку "Channel not found", даже если пользователь попытается угадать номер приватного канала.

---

## 3. Статистика (`/stats`)

Команда `/stats` выводит количество привязанных чатов, подписчиков и пересланных сообщений.

* **В личных сообщениях с ботом:**
  * **Владелец:** видит статистику по *всем* мостам в системе.
  * **Суб-админ:** видит статистику только по мостам, *созданным им лично*.
  * **Если Владелец не задан:** любой пользователь видит статистику по всем мостам (публичный режим).
* **Внутри Telegram-группы:**
  * Статистику конкретной группы может запросить Владелец, Суб-админ (создавший этот мост), либо **любой Администратор** этой Telegram-группы. Обычным участникам статистика недоступна.

---

**Итог и потенциальные места для улучшения:**

1. В "публичном режиме" (без Владельца) команда `/stats` в личке показывает все мосты системы любому пользователю, что может быть нежелательно с точки зрения приватности.
